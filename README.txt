Freight Elevator - Cadmium v2

This repository contains the Freight Elevator model implemented in Cadmium v2.

Repository structure
--------------------
- atomics/
  Atomic DEVS models (ECall, EControl, EVehicle).
- data_structures/
  Shared message and type definitions.
- input_data/
  Input files used by the experiment launchers.
- scripts/
  Windows launch scripts. Use the `.bat` files from PowerShell or Command Prompt.
- test/
  Stand-alone experiment frames for the atomic and coupled tests.
- top_model/
  Coupled models and top-level simulator.
- bin/
  Built executables and required Windows runtime DLLs after `make all`.
- build/
  Object files created by the build.
- simulation_results/
  CSV logs generated by the experiment scripts.

Expected clone layout
---------------------
Clone the repositories as sibling folders inside the same empty directory:

    <empty-folder>\
      cadmium_v2\
      FreightElevatorModel\

`cadmium_v2` and `FreightElevatorModel` must be beside each other.

Windows build prerequisites
---------------------------
Install MSYS2 and one of the gcc/make toolchains. The repository includes a local
`make.bat` wrapper that auto-detects a supported MSYS2 toolchain under `C:\msys64`.

Install MSYS2 from PowerShell:

    winget install -e --id MSYS2.MSYS2

Then open the MSYS2 UCRT64 shell and install the toolchain:

    pacman -S --needed mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-make

Fresh-clone Windows workflow
----------------------------
From an empty folder:

    git clone https://github.com/SimulationEverywhere/cadmium_v2 -b dev-rt
    git clone https://github.com/IshaqNour/FreightElevatorModel.git
    cd FreightElevatorModel

***Note: You can skip to "Run experiments" section if bin\ exists   

From Command Prompt:

    make clean
    make all

From PowerShell:

    .\make.bat clean
    .\make.bat all

If you prefer to call the real MSYS2 make executable directly, that also works:

    C:\msys64\mingw64\bin\mingw32-make.exe clean
    C:\msys64\mingw64\bin\mingw32-make.exe all

The build creates:
- `bin/freight_elevator_top.exe`
- `bin/ecall_test.exe`
- `bin/econtrol_test.exe`
- `bin/evehicle_test.exe`
- `bin/elevator_test.exe`
- runtime DLLs copied into `bin/`

Run experiments
---------------
After `make all`, run the experiment scripts from the repository root.

Run everything:

    .\scripts\run_all_experiments.bat

Or run by category:

    .\scripts\run_atomic_experiments.bat
    .\scripts\run_coupled_experiments.bat
    .\scripts\run_top_experiments.bat

The `.bat` files call PowerShell internally with `-ExecutionPolicy Bypass`, so they
work from both PowerShell and Command Prompt without extra setup.

Generated output files
----------------------
The scripts create these CSV files in `simulation_results/`:

- `A1_ecall.csv`
- `A2_ecall.csv`
- `A3_ecall.csv`
- `A4_evehicle.csv`
- `A5_econtrol.csv`
- `A6_econtrol.csv`
- `C1_elevator.csv`
- `C2_elevator.csv`
- `T1_top.csv`
- `T2_top.csv`

If bin/ is missing
------------------
If `bin/` was removed, rebuild first:

    make all
